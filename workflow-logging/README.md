# Конвенция: рабочий лог (stream + projects)

Plain text под Sublime — без markdown-разметки в логах. Один **месячный** stream-файл, внутри — разделители **дней**.

---

## Два уровня

| Уровень | Файл | Содержание | Живёт |
|---------|------|------------|--------|
| **Stream** | `stream/YYYY/YYYY-MM.txt` | Всё за месяц: задачи, мысли, команды | ~1 месяц, потом архив |
| **Projects** | `projects/<slug>.txt` | Отобранное: решения, шпаргалки, контекст | Годами, **коротко** (~150–250 строк) |

Stream можно засорять. В projects попадает только после **дистилляции** (раз в месяц, при смене stream-файла).

---

## Каталог

```
workflow/
├── README.md
├── stream/
│   └── 2025/
│       ├── 2025-05.txt
│       └── 2025-06.txt          # текущий месяц
├── projects/
│   ├── express-checks.txt
│   └── _personal.txt
├── archive/                     # старые stream (tar или как есть)
└── templates/
    ├── month-stream.txt
    └── project.txt
```

Расширение `.txt` — чтобы Sublime не вёл себя как с md (подсветка при желании через Plain Text).

## Stream: один файл на месяц

**Имя:** `stream/2025/2025-06.txt` (год-месяц).

**Новый день внутри файла** — разделитель (ваш формат, зафиксированный):

```
---[03.06.26]----------------------------------------------------------------------------------------------
```

Правила разделителя:

- Начало: `---[DD.MM.YY]`
- Дальше дефисы до **конца строки** (удобно визуально в Sublime; длина ~100 символов не критична, главное — единообразие).
- Пустая строка после разделителя — по желанию, но всегда одинаково (рекомендуется одна пустая строка).

Шапка месяца в файле: `=== 2025-06 ===` (не markdown, просто ориентир). Первый день — разделитель сразу под ней (см. шаблон).

Пишете **в конец файла**. Незакрытые дела естественно остаются внизу — отдельный inbox не нужен.

## Строка записи

Одна строка = одна мысль/задача (продолжение — следующие строки без типа, с отступом или как вам удобно в Sublime).

```
[express-checks] todo проверить dedup в exporter
[adis-infra] note staging kube context: cluster-foo
[—] cmd poetry install --only main
```

| Часть | Значение |
|-------|----------|
| `[slug]` | проект; `[—]` — без проекта |
| тип | см. ниже |
| остальное | текст |

### Типы (только три)

| Тип | Когда | После дистилляции |
|-----|--------|-------------------|
| **todo** | Нужно сделать | Сделано → забыть; повторяющееся/долгое → в projects в блок OPEN |
| **note** | Факт, решение, идея «пригодится позже» | Перенести в projects (DECISIONS / CONTEXT) |
| **cmd** | Команда, конфиг, фрагмент для копипаста | В projects в COMMANDS, если ещё актуально |

Другие типы не вводить — иначе снова перестанете ими пользоваться.

Время в строке **не пишем** — день уже задан разделителем выше.

## Ротация

### Когда новый stream-файл

**1-го числа календарного месяца** (или при первой записи нового месяца):

1. Создать `stream/YYYY/YYYY-MM.txt` из шаблона `templates/month-stream.txt`.
2. Пробежать **прошлый** месяц — дистилляция (ниже).
3. Старый файл не трогать (живой архив) или положить в `archive/`.

### Если месяц раздулся

| Условие | Действие |
|---------|----------|
| Файл > **~800–1000 строк** до конца месяца | Редко: продолжить в `2025-06-b.txt` |
| Конец месяца | Дистилляция обязательна |
| Конец года | По желанию zip `stream/2025/` в `archive/` |

### Чего избегать

- Один бесконечный `log.txt` на всё.
- Отдельный файл на каждый день — слишком много файлов.
- Отдельный файл stream на проект — каша размажется.

---

## Дистилляция (раз в месяц, ~20–40 мин)

При открытии нового `YYYY-MM.txt`:

1. Прочитать прошлый `stream/.../YYYY-MM.txt` (можно по разделителям дней).
2. Для каждой строки с `[slug]`:
   - **todo** выполнен → ничего не переносить
   - **todo** всё ещё актуален → `projects/<slug>.txt` секция OPEN
   - **note** ценная → DECISIONS или CONTEXT (одна строка + `из stream 2025-05, 12.05`)
   - **cmd** нужен дальше → COMMANDS
   - шум / устарело → пропустить
3. Пробежать `projects/*`: вычистить неактуальное, слить дубли.

Важное должно оказаться в projects **в том же месяце**, иначе считается несущественным.

## Projects: `projects/<slug>.txt`

Plain text, фиксированные секции (заголовки строками — хорошо видно в Sublime):

```
=== express-checks ===

--- DECISIONS ---
2025-05-12 journal dedup по check_id (stream 2025-05)

--- COMMANDS ---
poetry run pytest --cov=src

--- CONTEXT ---
сервисы: worker, exporter, api

--- OPEN ---
todo вынести workflow config в chart
```

Секции не плодить. Перенос из stream — **короткая строка**, не копия простыни.

## Проекты

| Вопрос | Ответ |
|--------|--------|
| Сколько файлов? | Один `.txt` на slug + `_personal.txt` |
| Новый проект? | Скопировать `templates/project.txt` |
| Забыли `[slug]`? | При дистилляции → `_personal.txt` |

Slug: короткий, латиница: `express-checks`.

## Sublime (по желанию)

- Сниппет на разделитель дня: вставляет `---[DD.MM.YY]` + дефисы до 100 колонки.
- Сниппет на строку: `[${1:slug}] ${2:todo} $0`
- Scope: Plain Text для `stream/` и `projects/`

---

## Поиск и прочее

- `rg "dedup" ~/tmp/workflow/projects/` — часто
- `rg "dedup" ~/tmp/workflow/stream/` — реже, по архиву месяцев
- Секреты не писать; только «в vault, id X»
- В ИИ — `projects/<slug>.txt`, не месячный stream
- Бэкап всего каталога `workflow/` (git private / syncthing)
